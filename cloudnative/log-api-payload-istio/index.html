Log API Payload in istio

2020-04-11

Overview⌗

API Payload capture is a key tool in troubleshooting.

tcpdump is the tool working at ISO layer 3~6, Our api working working at Layer 7(HTTP), many capture rules are work only on Layer 7. E.g. filter payload by URL Path is hard.

tcpdump -i enp0s8 -s 0 -v -n -l | egrep -i "POST /|GET /|Host:"

In Istio(ADP Service Mesh), Envoy is the sidecar. It supports HTTP Filters, It work on Layer 7, We can create some filter on Envoy to capture payload.

tcpdump on minikube(FT) or Node local⌗ 

[root@iam-iam-cloudnative-common-ft-200407t182432-a2 ~]# ps -ef | grep -i permission
